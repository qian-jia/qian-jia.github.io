Build a Personal Website with Github Pages and Github Actions

Published on: April 2, 2023 | Reading Time: 4 min | Last Modified: April 2, 2023

website

GitHub Pages is a powerful tool for building and hosting static websites, while GitHub Actions allow you to automate your workflows.

When combined, they provide an excellent platform for building and deploying your website with ease. In this article, I will show you how did I use GitHub Pages and GitHub Actions to build this personal website.

Step 1: Create Repositories

The first step is to create two new repositories on GitHub. I use one of them as the GitHub page repository (personal website) to store my static website. This one should be named username.github.io. I use the other one as the GitHub action repository (website generator) to automatically build and deploy the website.

To do this, go to your GitHub account and click on the “New” button. Give your repository a name and a description, and make sure that the GitHub page repository is set to public and the GitHub action repository is set as a private one. You can choose to add a README file, but it’s not necessary. Each repository only has a main branch.

The reason I use two repositories instead of one is that it allows me to store the original website content in a private environment.

Step 2: Create the Website

Once your repositories is set up, you can create your website. There are many ways to create a static website. For example, we can install a static site generator like Jekyll, Hugo, Hexo, or Gatsby. These tools allow you to create a website quickly and easily, using templates and themes.

In my case, after I created a website, I got website documents shown in the following picture. I uploaded these files to the GitHub action repository.

A simple way that I recommend here is to download website components or the example site of a static website theme (e.g., a Hugo website theme) directly from GitHub and upload them to the GitHub action repository we created. In this way, you don’t need to download and install any static site generator. We can directly modify the uploaded website files on GitHub and automatically generate our own website with GitHub actions.

Step 3: Enable GitHub Pages

To enable GitHub Pages for your website, go to the settings tab of your repository, and scroll down to the Code and automation section. Under “Pages—>Source”, select deploy from a branch, then select the branch that contains your static website files (usually main or master).

Your website should now be live at username.github.io. If you go to this URL, you should see your website.

Step 4: Set Up a GitHub Action to Deploy Your Website

The next step is to set up a GitHub Action to automatically build and deploy your website whenever you push changes to your repository. To do this, you’ll need to create a new file in your repository called .github/workflows/main.yml. This file will contain the instructions for your GitHub Action.

Here’s an example of what your main.yml file might look like when you use Hugo:

name: GitHub Pages

on:
  push:
    branches:
      - main  # Set a branch name to trigger deployment
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-22.04
    permissions:
      contents: write
    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive)
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.111.3'

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        # If you're changing the branch from main,
        # also change the `main` in `refs/heads/main`
        # below accordingly.
        if: ${{ github.ref == 'refs/heads/main' }}
        with:
          deploy_key: ${{ secrets.ACTIONS_DEPLOY_KEY }} 
          # if you use a different secret name, change "ACTIONS_DEPLOY_KEY" to your secret name. 
          # I will show how to setup the deploy key and secret in Step 5.
          external_repository: username/username.github.io # change the username!!!
          publish_branch: main  # default: gh-pages
          publish_dir: ./public

This GitHub Action will run whenever you push changes to the main branch of the GitHub action repository. It will check out your code, build your website using Hugo (or your chosen static site generator), and deploy it to the main branch of the GitHub page repository.

Step 5: Set Up Your Deploy Key1

Generate your deploy key with the following command.

ssh-keygen -t rsa -b 4096 -C "your github account email address" -f gh-pages -N ""

You will get 2 files:

  • gh-pages.pub is a public key
  • gh-pages is a private key

Next, Go to Repository Settings

  • Go to Deploy Keys (the GitHub page repository) and add your public key with the Allow write access
  • Go to Secrets (the GitHub action repository) and add your private key as ACTIONS_DEPLOY_KEY

Your personal website is done! You can access it using the URL in the format https://<username>.github.io/, where <username> is your GitHub username.
